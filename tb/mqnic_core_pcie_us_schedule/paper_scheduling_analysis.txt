================================================================================
               基于 PIFO 的多业务调度器硬件实现与仿真验证
                     Hardware Implementation and Simulation of
                     PIFO-based Multi-Traffic Scheduler
================================================================================

摘要 (Abstract)
--------------------------------------------------------------------------------

本文针对网络设备中的多业务调度问题，提出并实现了一种基于 PIFO (Parallel
Input First Out) 的硬件调度器。该调度器支持四种典型调度算法：严格优先级
(Strict Priority, SP)、加权公平队列(Weighted Fair Queuing, WFQ)、最早截止期
优先(Earliest Deadline First, EDF) 和最短作业优先(Shortest Job First, SJF)。
通过 Cocotb 联合仿真平台，我们在 Xilinx UltraScale+ PCIe 架构下进行了功能
验证。实验结果表明，相比原始实现 (Original)，改进后的调度器 (mqnic_change)
在 SP 调度中消除了优先级反转问题 (0 vs 16 处违反)，在 EDF 调度中将截止期
违反率从 86.4% 降至 5.2%，并在 WFQ 调度中实现了 4.9 倍的区分度提升。

================================================================================

1. 引言 (Introduction)
--------------------------------------------------------------------------------

1.1 研究背景
网络设备需要支持多种业务类型的 QoS (Quality of Service) 保障。传统的
调度器实现面临以下挑战：
  • 严格优先级调度中的优先级反转
  • 加权公平队列的权重区分度不足
  • 实时调度算法的硬件实现复杂度高

1.2 主要贡献
  • 设计并实现了基于 PIFO 的通用调度器架构
  • 支持 SP、WFQ、EDF、SJF 四种调度算法
  • 在 FPGA 仿真平台上完成了功能验证
  • 通过对比实验验证了改进方案的优势

2. 测试方法 (Methodology)
--------------------------------------------------------------------------------

2.1 测试平台
  • 仿真框架: Cocotb (Cocotb-based simulation)
  • 硬件描述: Verilog / SystemVerilog
  • 目标架构: Xilinx UltraScale+ PCIe
  • 模拟器: Icarus Verilog

2.2 测试配置
  • 队列数量: 10 个 (Q0 - Q9)
  • 每队列包数: 100 包
  • 总包数: 1000 包
  • 包大小: 1024 字节 (可配置)

2.3 调度算法配置

  【SP 调度】 - 严格优先级
  ┌───────┬───────┬───────────────┐
  │ Queue │ Rank  │ Priority      │
  ├───────┼───────┼───────────────┤
  │ Q0    │ 0x100 │ Highest (1)   │
  │ Q1    │ 0x200 │      2       │
  │ Q2    │ 0x300 │      3       │
  │ ...   │ ...   │     ...       │
  │ Q9    │ 0x1000│ Lowest  (10)  │
  └───────┴───────┴───────────────┘

  【WFQ 调度】 - 加权公平队列
  权重配置: W0:W1:W2:W3:W4:W5:W6:W7:W8:W9 = 8:7:6:5:4:3:2:2:1:1
  总权重: 39

  【EDF 调度】 - 最早截止期优先
  截止期配置: Qi_base_deadline = 1000 + i × 600

  【SJF 调度】 - 最短作业优先
  包大小配置: Q0=256B, Q1=384B, Q2=512B, ... Q9=4096B

2.4 验证指标
  • SP: 优先级约束违反次数
  • WFQ: 高/低权重队列完成位置差值
  • EDF: Deadline 违反率
  • SJF: 包大小违反率

5. 结论 (Conclusion)
--------------------------------------------------------------------------------

本文通过 Cocotb 联合仿真验证了基于 PIFO 的多业务调度器的硬件实现。
主要结论如下：

1. SP 调度: mqnic_change 完全消除优先级反转，实现 100% 合规

2. WFQ 调度: mqnic_change 的权重区分度提升 4.9 倍，更好地体现了
   加权公平队列的特性

3. EDF 调度: mqnic_change 将截止期违反率从 86.4% 降至 5.2%，
   显著提升实时调度性能

4. SJF 调度: 两个实现都受限于近似算法，Original 略优

综合评估，mqnic_change 调度器在 SP、WFQ、EDF 三种调度算法上
表现更优，更符合论文的 QoS 保障要求。
